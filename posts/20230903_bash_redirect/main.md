# Bashのリダイレクトに関するメモ

（書きかけです。）

　連載のための調べものです。

## 矢印の種類

| 種類             | 文字                     |
|:-----------------|:-------------------------|
| >, < | 書き出し/読み込み |
| >>, << | ファイルへの追記/ヒアドキュメント |
| <<< | ヒアストリング |
| <> | 読み書き |
| `>|` | -Cオプションが設定されているときにファイルを上書き |

### <>の使い方

[シェル・ワンライナー160本ノック](https://amzn.to/3P0UxaS)に、次のような利用例が示されている。

```bash
$ exec 3<> /dev/tcp/f.ueda.tech/80        #このBashのプロセスのFD3を読み書きモードで開く
$ echo -ne "GET /eki/ HTTP/1.0\\n\\n" >&3 # http://f.ueda.techにリクエストを送信
$ cat <&3                                 #送られてきたデータを表示
HTTP/1.1 200 OK
Server: nginx/1.19.6
（略）
<!DOCTYPE html>
（略。HTMLのデータが出力される）
```

## &の使い方

| 種類             | 文字                     | 例 |
|:-----------------|:-------------------------|-----|
| &n | n番のファイル記述子へコピー | >&2, >>&2, <4 |

### `<&n`の使い方

```
$ rev 9<<<abc <&9
cba
```
